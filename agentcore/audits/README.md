# Audits

Use this folder for knowledge-base health checks.

## Audit Checklist

- Stale claims that newer evidence may supersede.
- Contradictions between pages.
- Important claims without source links.
- Orphan pages with no inbound links from `index.md` or related pages.
- Broken relative links.
- Missing project status, decisions, outputs, or next actions.
- Open blockers that are no longer relevant.

Record each audit as a dated Markdown file and append the result to `../log.md`.
